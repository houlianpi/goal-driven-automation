##Overview
Smoke Testing is split into two sub-tasks:

- **Sanity Testing**: Quick Sanity testing of the build and external build.
- **Smoke Testing**: More extensive Smoke testing of the build across a matrix of supported platforms.
- **General Testing**: General testing of the build for scenarios that are difficult to automate.

##Daily workflow

1. Wait for the "Edge build <build_number> is ready for Sanity Testing and Smoke Testing" e-mail.
2. Complete **Sanity Testing** validation for the build.
   - Once completed, reply all to the e-mail reporting the outcome of the validation.
   - In your reply add Edge Smoke Results edgesmokeresults@microsoft.com to the CC line.
   - If during testing you experienced a crash on launch please include Edge Reliability Sheriff Help Desk <EdgeRelHelp@microsoft.com> in the TO line and make sure you attach to the bug any *.dmp files found in `.\Users\<user>\AppData\Local\Microsoft\<Edge Channel Folder>\User Data\Crashpad\reports`.  
   - If you encounter crash/hang during the "2. External Build Sanity Test" testing for Windows. Get the call stacks by following steps mentioned [here](https://dev.azure.com/microsoft/Edge/_wiki/wikis/Edge.wiki?wikiVersion=GBwikiMaster&_a=edit&pagePath=%2FTeams%2FDeployment%2FFlighting%2FSmoke%20Testing&pageId=143&anchor=steps-to-get-the-call-stack-for-external-builds-crash%2Fhang-dumps-for-windows%3A) and update in the bug.
3. Complete **Smoke Testing** validation.
   - Once completed send results report to Edge Smoke Results edgesmokeresults@microsoft.com. 
   - In your report, include the two tables regarding mini installers and differential installers from the notification email.
   - If during testing you experienced a crash on launch please include Edge Reliability Sheriff Help Desk <EdgeRelHelp@microsoft.com> in the TO line and make sure you attach to the bug any *.dmp files found in `.\Users\<user>\AppData\Local\Microsoft\<Edge Channel Folder>\User Data\Crashpad\reports`.

4. Complete **General Testing** validation.
   - Once completed send results report to Edge Smoke Results edgesmokeresults@microsoft.com.

###Troubleshooting
If you don't receive the "Edge build <build_number> is ready for Sanity Testing and Smoke Testing" e-mail do the following:
- See if [Edge-SmokeTest](https://dev.azure.com/microsoft/Edge/_release?_a=releases&view=mine&definitionId=68) actually triggered but ADO just failed to send the e-mail. If that is the case continue as usual.
- See if [edge-official-promotion](https://dev.azure.com/microsoft/Edge/_build?definitionId=40510) build executed and succeeded. If it didn't execute or it failed please look for status of issues in this team channel: [Anaheim - Branch Sheriff Office](https://teams.microsoft.com/l/channel/19%3a46fc413b05ba495f84a24fe8ea92bdee%40thread.skype/Anaheim%2520-%2520Branch%2520Sheriff%2520Office?groupId=0183ae9f-67b8-4ab0-80bf-761036ec893d&tenantId=72f988bf-86f1-41af-91ab-2d7cd011db47) and do send e-mail to Edge Smoke Results edgesmokeresults@microsoft.com to let us know too.
- Whenever there is a build generation failure for any platform, please send an email to BDW DRI <bdwdri@microsoft.com> and copy to Edge Smoke Results <edgesmokeresults@microsoft.com> and Edge Smoke Testers <edgesmoke@microsoft.com> with failure snapshots.

**Note**:
When build installers are not available for all or one of the platforms (win / arm / mac) for a build then please perform the sanity & smoke test on previous build of same version if all the installers are available. This is possible only when we have multiple builds on same day if same version or on Monday.
**_Example_**_: Say, build installers are not available for the platforms (arm / mac) for a build 86.0.576.0 then we need to perform sanity & smoke test on previous build 86.0.575.0 if all the installers are available.

**External Build Sanity Case**: Any issue related to this case, assign the bug to krisk@microsoft.com

###Installing Edge

####Windows
This is how to update or install Edge for a particular **channel** in a machine:
1. **Update**: On a machine where you previously bootstrap installed Edge for a particular **channel** from [Channel Download Page](https://edgeesportal.azurewebsites.net/bootstrapper_releases/index.html) you launch Edge for that **channel** and open the "About Microsoft Edge" page to check for updates.
2. **Bootstrapper Install**: On a clean machine you install Edge for a particular **channel** from [Channel Download Page](https://edgeesportal.azurewebsites.net/bootstrapper_releases/index.html).
3. **MiniInstaller Install**: On a clean machine you install Edge for a particular **channel** by downloading the installer as instructed in [Downloading Edge](https://microsoft.visualstudio.com/Edge/_wiki/wikis/Edge.wiki/143/Smoke-Testing?anchor=downloading-edge) and running the installer from a **user command line** window as follow:
--- **Canary**: `"<installer> --msedge-sxs --verbose-logging"`
--- **Dev**: `"<installer> --msedge-dev --system-level --verbose-logging"`
--- **Beta**: `"<installer> --msedge-beta --system-level --verbose-logging"`
--- **Stable**: `"<installer> --system-level --verbose-logging"`
4. **Differential Update**: On a machine where you previously installed Edge for a particular **channel** and particular **old-build** you can update it to a **new-build** by downloading the differential installer for **old-build**->**new-build** as instructed in [Downloading Edge](https://microsoft.visualstudio.com/Edge/_wiki/wikis/Edge.wiki/143/Smoke-Testing?anchor=downloading-edge) and running the installer from a **user command line** window as follow: 
--- **Canary**: `"<differential_installer> --msedge-sxs --verbose-logging"`
--- **Dev**: `"<differential_installer> --msedge-dev --system-level --verbose-logging"`
--- **Beta**: `"<differential_installer> --msedge-beta --system-level --verbose-logging"`
--- **Stable**: `"<differential_installer> --system-level --verbose-logging"`
5. **Uninstall**: Right click install point in Start menu and click **Uninstall**:

    Find program in Programs and Features and click **Uninstall**:
